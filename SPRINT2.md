# Sprint 2 — Web-сервис трансформации ВОР → ЛЗК (Flask)

**Дата:** 02.04.2026
**Статус:** Завершён

---

## Цель спринта

Добавить второй вариант использования инструмента — **web-сервис** на Flask, доступный через браузер. Без авторизации, без хранения файлов на сервере. Загрузил → настроил → скачал.

---

## Выполнено

### Рефакторинг архитектуры

- [x] Выделен `vor_core.py` — ядро трансформации без GUI и дисковых зависимостей
- [x] `transform_vor.py` переведён в тонкую обёртку: импортирует из `vor_core`, содержит только GUI-код
- [x] Сигнатура `transform()` расширена: принимает `BytesIO` на входе, возвращает `BytesIO` при `output_target=None`

### Web-сервис (`app.py`, порт 6511)

- [x] `GET /` → одностраничный UI
- [x] `POST /detect` → JSON: список листов, авто-определённый лист, авто-определённые колонки Кол-во
- [x] `POST /transform` → Excel-файл (download) — вся обработка в памяти, диск не задействован
- [x] Лимит загрузки 50 МБ
- [x] Обработка ошибок с возвратом JSON

### Web-интерфейс (`templates/index.html`)

- [x] Drag-and-drop загрузка файла
- [x] Авто-вызов `/detect` при выборе файла
- [x] Авто-подстановка листа и колонки «Кол-во итого»
- [x] Ручная корректировка всех четырёх колонок
- [x] Скачивание результата через `URL.createObjectURL()` (без перезагрузки страницы)
- [x] Статусные сообщения и блокировка кнопки при обработке
- [x] Vanilla JS, без сторонних фреймворков

### Прочее

- [x] `requirements.txt`: `flask>=3.0`, `openpyxl>=3.1`
- [x] `README.md` дополнен: инструкция для web-сервиса, обновлённая структура проекта

## Известные ограничения

- Development-сервер Flask не предназначен для продакшн-нагрузки; для деплоя использовать gunicorn/uvicorn
- Прогресс обработки на web-стороне не отображается (нет streaming)
