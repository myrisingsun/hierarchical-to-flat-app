# Sprint 1 — Базовый инструмент трансформации ВОР → ЛЗК

**Дата:** 02.04.2026
**Статус:** Завершён

---

## Цель спринта

Разработать универсальный GUI-инструмент, который трансформирует иерархический список ВОР (Ведомость объёмов работ) в плоский список ЛЗК (Лимитно-заборная карта) в формате Excel.

---

## Выполнено

### Ядро (логика трансформации)

- [x] Парсинг трёхуровневой иерархии (`раздел → подраздел → работа → материалы`)
- [x] Авто-определение строки начала данных (пропуск шапки произвольной высоты)
- [x] Авто-определение нужного листа по паттерну иерархических номеров (`1`, `1.1`, `1.1.1`)
- [x] Авто-определение колонки «Кол-во итого» по заголовкам (ключевые слова: итого, расход, кол-во)
- [x] Два режима работы:
  - **С материалами** — строки с `col_A = None` под каждой работой (→ один материал = одна строка)
  - **Только работы** — нет строк материалов (→ одна работа = одна строка, столбец материала пуст)
- [x] Остановка обработки при выходе за пределы иерархии (сводные таблицы и т.п.)
- [x] Формирование выходного Excel с заголовком, форматированием, заморозкой строки

### GUI (tkinter)

- [x] Диалог выбора входного файла (фильтр `.xlsx`)
- [x] Выпадающий список листов с авто-выбором найденного
- [x] Настройка колонок вручную (буква A–Z)
- [x] Выбор пути выходного файла
- [x] Прогресс-бар во время обработки (фоновый поток)
- [x] Предложение открыть результат по завершении
- [x] Отображение режима в строке статуса
- [x] Строка копирайта

### Исправления в ходе спринта

- [x] Исправлена ошибка `NameError: cannot access free variable 'e'` (Python 3.13 — захват переменных в lambda через аргументы по умолчанию)

## Известные ограничения

- Поддерживается иерархия до 3 уровней в стандартном варианте (раздел / подраздел / работа); более глубокие уровни трактуются как работы
- Колонки A–Z (одиночная буква); двухбуквенные индексы (AA, AB...) поддерживаются в коде, но не тестировались
